## rules → `.claude/rules/` (native path-scoped rules)

- **Why / Findings:** the security/architecture invariants and workflow conventions exist in C3 and CLAUDE.md prose but never auto-load PATH-SCOPED; thin `.claude/rules/*.md` make each one surface the moment Claude opens a governed file, with C3 kept as the source of truth — repo scan: present-weak — 5 enforced rules + 5 refs under `.c3/`, a rich `CLAUDE.md`, ESLint import-x zones + `scripts/check-dep-direction.mjs`; but NO native `.claude/rules/*.md`, and the `c3` CLI is not on PATH, so nothing surfaces a rule when a matching file is opened. Considered: full-duplication-from-c3, workflow-conventions-only, skip-rely-on-c3-and-CLAUDE.md.
- **Apply action:** CREATE — `.claude/rules/` is missing; create one `<name>.md` per `concerns.rules.files[]` entry. Each file is a THIN pointer to the canonical `.c3/rules/` rule of the same name (C3 stays source of truth).
- **Target:** the `.claude/rules/` directory (one `<name>.md` per `concerns.rules.files[]` entry)
- **Inputs:** `concerns.rules.files[]` (each `{name, paths?, obey[], avoid[], ref}`)
- **Steps:**
  1. `.claude/` is gitignored repo-wide (`.gitignore` has `.claude/`). Un-ignore the rules dir so the harness is tracked: change `.claude/` → `.claude/*` and add `!.claude/rules/` (git cannot re-include a child of an excluded directory, so the parent glob must switch to `.claude/*`).
  2. For each entry in `concerns.rules.files[]`, create `.claude/rules/<name>.md`.
  3. If the entry has `paths`, add YAML frontmatter listing each glob; if it has no `paths`, write no frontmatter (always-on rule — applies to `workflow-conventions`).
  4. Add a top **"NEVER"** list — one bullet per `avoid[]` item.
  5. Add sections phrased from `obey[]`, plus a line linking the canonical `.c3/rules/<...>.md` from `ref`.
- **Verify:** every `files[]` entry has a matching `.claude/rules/<name>.md`; each scoped entry's frontmatter `paths` matches the profile; every `avoid[]` item appears in its NEVER list; `git check-ignore .claude/rules/domain-purity.md` returns nothing (tracked).
