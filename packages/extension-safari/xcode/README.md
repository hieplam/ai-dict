# Xcode iOS Wrapper — AI Dictionary Safari Extension

This directory holds the iOS app wrapper that hosts the Safari Web Extension.
The actual Xcode project (`.xcodeproj`) is generated by
`xcrun safari-web-extension-converter` on a macOS machine with Xcode installed.

## First-time setup (macOS + Xcode required)

```bash
# 1. Build the web extension dist
pnpm --filter @ai-dict/extension-safari build

# 2. Generate the Xcode project (iOS-only, no macOS target)
xcrun safari-web-extension-converter packages/extension-safari/dist \
  --project-location packages/extension-safari/xcode \
  --app-name "AI Dictionary" \
  --bundle-identifier com.ai-dict.safari \
  --ios-only --no-open --no-prompt --copy-resources

# 3. Commit the generated xcode/ tree
git add packages/extension-safari/xcode
git commit -m "feat(extension-safari): add generated Xcode project"
```

## Syncing dist/ after a rebuild

After changing the web extension code:

```bash
pnpm --filter @ai-dict/extension-safari build
pnpm --filter @ai-dict/extension-safari xcode:sync
```

The `xcode:sync` script copies `dist/` into
`xcode/AI Dictionary Extension/Resources/` so you can build in Xcode
without re-running the converter.

## Structure (after generation)

```
xcode/
├── AI Dictionary.xcodeproj/    ← generated Xcode project (iOS target only)
├── AI Dictionary/              ← iOS host app source
│   ├── AppDelegate.swift
│   ├── SceneDelegate.swift
│   └── ...
├── AI Dictionary Extension/    ← Safari Web Extension target
│   ├── SafariWebExtensionHandler.swift
│   ├── Info.plist
│   └── Resources/              ← synced from dist/ by xcode:sync
│       ├── manifest.json
│       ├── sw.js
│       ├── content.js
│       ├── options.js
│       └── options.html
└── sync-dist.sh                ← sync script
```

## Release

Bundle 07 (`ci-release`) runs `build` → `xcode:sync` → `xcodebuild archive`
on the macOS runner and rewrites `MARKETING_VERSION` in the Xcode project
alongside the manifest `version` field.
